Logic Explanation

Array Logic
The primary array stores objects for the loot that's been added by the form. Those objects are constructed from a custom class that include four separate fields for the name, quantity, the value, and the rarity number. The class also provides a couple of features for getting the value and the name of the quality/rarity integer, which is useful when looping through each one of them. I'd planned to include some checks within the class in a future revision, to prevent inputting invalid values upon construction.

The quantity was added in the second phase, and itself was just a simple addition to the constructor. I had to also make sure that the loot table accounted for this since the quantity is now used in the value calculation.

updateUI() Logic
This in turn calls two additional functions to calculate the loot split and validation of the party size, and then enables or disables the button for splitting loot. This button isn't necessary anymore, but was left in as part of the assignment requirements.

Invalid State Logic
For Party Size and Item Quantity, a function is called that forces the value to be a nonzero positive integer. This also returns a result for displaying appropriate error messages, if desired. For Item Value some simple checks are done just to ensure it's a positive number. The absolute value is taken. If it's not a number then the value defaults to zero.

There is a handler for invalid party size entries, but with the way the logic is written that handler should never occur. But with how weird JavaScript can be and people possibly fiddling with developer tools in the browser it's better safe than sorry.

The button below will call a method to instantly load up the loot list table with random pieces of dungeon loot. Name, quantity, value, and rarity are all randomized. Useful for quickly testing certain features that doesn't involve manually entering items.
